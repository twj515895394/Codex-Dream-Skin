# Image Analysis Guidance

## Purpose

Analyze a reference image before creating a Codex Dream Skin. The goal is not to copy the image directly, but to transform its visual language into a long-term usable coding workspace.

## Analyze These Dimensions

### Subject

Identify:

- primary subject;
- visual focus;
- emotional meaning;
- whether the subject should remain visible or become inspiration only.

### Composition

Evaluate:

- available quiet area for Codex native heading and controls;
- subject position;
- balance between empty space and detail;
- whether the image can support both home and task routes.

Default preference:

- important subject on center-right;
- calm low-detail area on left;
- avoid placing high contrast elements behind text.

### Color Language

Extract:

- dominant colors;
- accent colors;
- shadow colors;
- highlight colors;
- emotional temperature.

Convert photography colors into UI-friendly palette:

- reduce saturation;
- soften extreme contrast;
- keep readable text contrast;
- prefer calm colors for long sessions.

### Mood Translation

Map the original feeling into workspace language.

Examples:

Family photo:

- warmth
- companionship
- creativity
- calm

Mountain photo:

- focus
- exploration
- clarity

Cyberpunk image:

- energy
- future
- technology

## Output

Before generating artwork, produce:

```text
Theme direction:
Mood:
Color palette:
Composition:
Left safe area:
Subject placement:
Long-session comfort considerations:
```

## Important

Do not generate a screenshot of Codex. Generate a clean wallpaper suitable for Codex native UI overlay.
